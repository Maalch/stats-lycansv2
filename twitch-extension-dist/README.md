# Lycans Tracker — Twitch Extension

A Twitch Extension for [lycanstracker.fr](https://www.lycanstracker.fr) that surfaces werewolf game statistics directly on Twitch streams.

## Components

| File | Type | Description |
|---|---|---|
| `panel.html` | Panel (318px) | Full player card below the stream |
| `video_component.html` | Video overlay | Toggleable badge in the bottom-right corner |

## Features

### Panel
- **Auto-detection:** Matches the broadcaster's Twitch login against `joueurs.json` — 83/89 known players are detected automatically. Others get a manual dropdown selector.
- **Player card:** avatar, primary title (🦎 Le·a Caméléon), win rate, games played, current streak
- **Camp performance:** animated bars for Villageois / Loup / Autres win rates
- **Achievements:** number of unlocked levels + top 3 highest-tier achievements with tier colour
- **Recent games:** last 5 games with outcome, map, and role
- **Dashboard link:** deep link to `lycanstracker.fr` with the player pre-selected

### Video Overlay
- **Collapsed:** small pill showing title emoji + win rate
- **Expanded:** mini card with name, title, win rate, games, and streak
- Toggle on click; preference saved in `localStorage`
- Fully transparent background — clicks pass through to the video except on the badge

## Architecture

```
Twitch Extension/
├── panel.html               Panel view
├── video_component.html     Video overlay view
├── shared/
│   ├── constants.js         Base URLs, cache TTLs, role emojis, tier metadata
│   ├── data-fetcher.js      Fetch + sessionStorage caching for all data files
│   └── player-matcher.js    Map Twitch channel name → joueurs.json → Steam ID
├── panel/
│   ├── panel.js             Panel rendering logic
│   └── panel.css            Panel styles (318px, dark Twitch theme)
└── overlay/
    ├── overlay.js           Overlay badge logic
    └── overlay.css          Overlay styles (transparent, pointer-events pass-through)
```

## Data sources

All data is fetched from static files on GitHub Pages — no backend required. CORS is open (`*`) by default on GitHub Pages.

| File | URL | Updated |
|---|---|---|
| `joueurs.json` | `https://www.lycanstracker.fr/data/joueurs.json` | Rare / manual |
| `playerSummary.json` | `https://www.lycanstracker.fr/data/playerSummary.json` | Weekly (Sunday 6 AM UTC) |
| `playerTitles.json` | `https://www.lycanstracker.fr/data/playerTitles.json` | Weekly |
| `playerAchievements.json` | `https://www.lycanstracker.fr/data/playerAchievements.json` | Weekly |

`playerSummary.json` is generated by `scripts/data-sync/generate-player-summary.js`. It is a compact (~100 KB) file with per-player quick stats, avoiding the need to download the full 15 MB `gameLog.json` inside an extension iframe.

## Generating the player summary locally

```bash
# From project root
npm run generate-summary
```

Outputs to `data/playerSummary.json` and copies to `public/data/`.

## Local development with the Developer Rig

1. Install the [Twitch Developer Rig](https://github.com/twitchdev/developer-rig) and run `yarn start` in its folder (serves from `https://localhost.rig.twitch.tv:3000`).
2. Create a new extension project in the rig pointing to this `Twitch Extension/` folder.
3. Add Panel and Video Component views.
4. Navigate to **Project Overview → Host with Rig**.
5. In **Extension Views**, you should see both the panel and overlay.

The `<script src="https://localhost.rig.twitch.tv:3000/twitch-ext.js">` tag in each HTML file is the rig shim — it provides mock `window.Twitch.ext` callbacks. This tag only works locally; the line is harmless in production (it 404s silently) because the production SDK is loaded from `extension-files.twitch.tv` first.

**Testing auto-detection:** The rig provides a fake `onContext` with a configurable channel name. Set it to a known player login (e.g. `ponce`) to verify the auto-match flow. Use an unknown name to test the manual selector fallback.

## Building for upload

```bash
npm run build-extension
```

This creates `twitch-extension-dist/` and, if PowerShell is available, `twitch-extension.zip` ready for upload to the Twitch Developer Console.

## Deployment (Twitch Developer Console)

1. Upload `twitch-extension.zip` in **Files → Hosted Test**.
2. Under **Capabilities**, ensure the extension is registered as both a **Panel** and a **Video Component**.
3. Add `https://www.lycanstracker.fr` and `https://static-cdn.jtvnw.net` to the **Content Security Policy** allowlist.
4. Test in hosted mode, then submit for review.

## Continuing sandbox development in the rig

This readme details how to use the [Developer Rig](https://github.com/twitchdev/developer-rig) to continue development on your extension. You can read about how to get started with the project [here](https://github.com/twitchdev/developer-rig#twitch-extensions-developer-rig).

Prerequisites:
- You have retrieved this sample code from the sandbox by selecting to download it within the UI.
- Developer Rig is installed locally and is running on your machine. Within the rig folder, run `yarn start`. The rig will be hosted on https://localhost.rig.twitch.tv:3000.

These steps will assume downloaded code is at `/Users/twitch/Downloads/sandbox-extension-hello-world` and that the rig is located at `/Users/twitch/developer-rig`. Your paths will differ per your username and folder structures.

1. In the rig running on your machine, create a new extension project.

- Extension Project Name: "Sandbox" (or whatever you like).
— Choose Extension: Create Local Extension
— Extension Types: Video Component, Panel
- Project Folder: /Users/twitch/Downloads/sandbox-extension-hello-world
- Add Code to Project: None (Just Create Project Folder)

2. Click the + button to add a new Panel view.

- Extension Type: Panel
- Viewer Type: Logged-Out Viewer

> Note: you can perform this same step for other extension views, such as components.

3. Navigate to Project Overview tab. Set Front-end Files Location to `.`. Click Host with Rig.

4. Navigate to Extension Views tab. You should see your extension!

## Differences between sandbox and rig

You may notice that the HTML and Javascript downloaded differs from what was being edited. A few things were added to help the extension runs in the rig:

- A script html element referencing `viewer.js` to load your javascript.
- A link html element referencing `style.css` to load styles.
- A script element `<script src="https://localhost.rig.twitch.tv:3000/twitch-ext.js"></script>`. This is required for local extensions.
- Two html files named `panel.html` and `video_component.html`. These are the naming conventions for these views for local extensions.
- Javascript code was updated to contain `twitch.onAuthorized` if it wasn't included. This function is required by extensions to load properly.
