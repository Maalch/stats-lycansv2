# Data Sync Architecture

## Overview

Stats Lycans uses a multi-source data pipeline with AWS S3 as the primary source. The system supports multiple teams through configuration-based filtering.

## Architecture

```
AWS S3 Bucket
    │
    └── StatsList.json (array of game log URLs)
          │
          ├── Ponce-20250116183045.json
          ├── Tsuna-20250117091522.json
          └── Nales-20250118102030.json
                │
                ▼
        fetch-data-unified.js
                │
                ├─[main]──► data/gameLog.json
                │            data/joueurs.json
                │            data/index.json
                │
                └─[discord]─► data/discord/gameLog.json
                             data/discord/joueurs.json
                             data/discord/index.json
```

## Key Files

| File | Purpose |
|------|---------|
| `fetch-data-unified.js` | Main AWS sync script (supports multiple teams) |
| `shared/data-sources.js` | Team configuration (prefixes, output dirs, filters) |
| `shared/sync-utils.js` | Core utilities (fetch, merge, validate) |
| `generate-rankings.js` | Generate player rankings from game data |
| `generate-titles.js` | Generate player titles based on stats |
| `generate-achievements.js` | Generate player achievements |

## Team Configuration

Located in `scripts/data-sync/shared/data-sources.js`:

```javascript
export const DATA_SOURCES = {
  main: {
    name: 'Main Team',
    outputDir: 'data',
    gameFilter: (gameId) => gameId.startsWith('Ponce-') || 
                           gameId.startsWith('Tsuna-') || 
                           gameId.startsWith('khalen-'),
    generateJoueurs: true,
  },
  discord: {
    name: 'Discord Team',
    outputDir: 'data/discord',
    gameFilter: (gameId) => gameId.startsWith('Nales-'),
    generateJoueurs: true,
  }
};
```

## Output Files

### Required Files (generated by sync)

| File | Description |
|------|-------------|
| `gameLog.json` | Unified game data with all games |
| `joueurs.json` | Player registry extracted from games |
| `index.json` | Metadata about the dataset |

### Generated by Separate Workflows

| File | Workflow | Schedule |
|------|----------|----------|
| `playerRankings.json` | `update-rankings-titles.yml` | Weekly (Sunday 6 AM UTC) |
| `playerTitles.json` | `update-rankings-titles.yml` | Weekly (Sunday 6 AM UTC) |
| `playerAchievements.json` | `update-rankings-titles.yml` | Weekly (Sunday 6 AM UTC) |

## Sync Behavior

### Incremental Sync (default)
- Loads existing `gameLog.json`
- Fetches only files from last 7 days (file-level)
- Updates games from last 6 hours (game-level)
- Skips older games that haven't changed

### Full Sync (`--full` flag)
- Ignores existing data
- Fetches all files listed in StatsList.json
- Rebuilds entire dataset from scratch

## Validation Pipeline

After each sync, the workflow validates:
1. Required files exist (`gameLog.json`, `index.json`, `joueurs.json`)
2. All files are valid JSON
3. Files are not empty
4. `joueurs.json` contains expected `Players` array

## Local Development

```bash
# Main team sync
npm run sync-data-aws

# Discord team sync
npm run sync-data-discord

# Full sync (ignore cache)
cd scripts/data-sync
node fetch-data-unified.js main --full
```

## Adding a New Team

1. Add configuration to `shared/data-sources.js`:
```javascript
newTeam: {
  name: 'New Team',
  outputDir: 'data/newteam',
  gameFilter: (gameId) => gameId.startsWith('NewPrefix-'),
  generateJoueurs: true,
  modVersionLabel: 'New Team - Multiple AWS Versions',
  indexDescription: 'Game logs for New Team.'
}
```

2. Run sync: `node fetch-data-unified.js newTeam`

3. Create workflow file: `.github/workflows/update-newteam.yml`

## Error Handling

The sync script handles errors gracefully:
- Individual file fetch failures don't stop the sync
- Corrupted games (missing `EndDate`) are skipped
- Games with fewer than 8 players are filtered out
- All errors are logged with context
